# ADR-0006: Fork the baseline into an owned, built config; decouple update-url

- **Status:** Accepted
- **Date:** 2026-07-08

## Context
The baseline `lazy.conf` is a **remote subscription** (`update-url` →
`johnshall.github.io/.../lazy.conf`). On refresh, Shadowrocket **overwrites the entire file**
with the upstream version — which would erase our strategy groups, providers, and regex. The
baseline is also not locally owned or versioned, blocking the project's engineering goals.

## Decision
- **Fork** the baseline: keep the original verbatim at `config/lazy.conf` (reference/rollback),
  and produce a separate, owned artifact `config/AI-Ultimate.conf` via the build.
- **Re-point `update-url`** to *this repo's own* raw config:
  `https://raw.githubusercontent.com/yomixiba0225/AI-Ultimate-Shadowrocket/main/config/AI-Ultimate.conf`
  so "Update" pulls our engineered config, not a third-party file.
- Treat the config as **configuration-as-code**: generated by `scripts/build.py`, never
  hand-edited.

## Consequences
- ✅ Local edits are durable; no silent overwrite from upstream.
- ✅ Updates become a normal git + push flow the user controls.
- ✅ Original baseline preserved for diffing and rollback.
- ⚠️ Users must import from the new URL/file; the upstream subscription is no longer the source.
- ⚠️ Remote rule-sets referenced inside the config still track upstream `master` (see ROADMAP
  v1.2.0 for optional commit-pinning, GAP G13).
